## Итерация 1 — Базовый Telegram-бот (polling, /start)
- [ ] Инициализировать проект: `uv`, `Makefile` (`install`, `run`, `clean`).
- [ ] Добавить `src/bot/main.py` с запуском `aiogram` (polling).
- [ ] Добавить хендлер `/start` → «Бот готов к работе».
- [ ] Настроить базовый `logging` и `config.py` (чтение `.env`).
- [ ] Протестировать: запустить `make run`, отправить `/start` → получить ответ.

## Итерация 2 — Ответы без LLM (эхо)
- [ ] Добавить `handlers.py` с обработкой текстовых сообщений.
- [ ] Отвечать пользователю тем же текстом (без LLM).
- [ ] Протестировать: отправить произвольный текст → получить эхо.

## Итерация 3 — Подключение LLM через OpenRouter
- [ ] Добавить `llm/client.py` (openai client с `base_url` OpenRouter).
- [ ] Добавить `llm/prompt.py` с системным промптом и сборкой сообщений.
- [ ] Заменить эхо на ответ LLM одним сообщением (без стриминга).
- [ ] Протестировать: задать вопрос → осмысленный ответ на русском.

## Итерация 4 — Файловый контекст и окно 20
- [ ] Добавить `storage.py` и `context.py` (файлы в `data/<chat_id>/`).
- [ ] Хранить и подмешивать последние 20 сообщений в промпт.
- [ ] Восстанавливать контекст после перезапуска.
- [ ] Протестировать: переписка, рестарт бота → контекст сохраняется.

## Итерация 5 — Суммаризация каждые 20 и слияние
- [ ] Добавить `llm/summarizer.py` (краткая русская сводка 5–8 строк).
- [ ] После каждых 20 сообщений: суммаризация и слияние с предыдущей.
- [ ] Подмешивать сводку в промпт при генерации ответа.
- [ ] Протестировать: временно установить `WINDOW_SIZE=3` → проверить работу.

## Итерация 6 — Команды управления
- [ ] `/reset`: очистка истории и суммаризации чата.
- [ ] `/status`: показать счётчики и наличие суммаризации.
- [ ] Протестировать: выполнить обе команды, проверить поведение.

## Итерация 7 — Ошибки и минимальные ретраи
- [ ] Обернуть LLM-запрос минимальным ретраем при сетевых ошибках (1 раз, опционально).
- [ ] При сбое отвечать коротким сервисным сообщением и логировать ошибку.
- [ ] Протестировать: выключить сеть/ключ → убедиться в корректном фолбэке.

## Итерация 8 — Документация и конфигурация
- [ ] Обновить `README.md` (запуск, переменные окружения).
- [ ] Добавить/актуализировать `.env.example`.
- [ ] Протестировать: чистая установка по README → бот отвечает.
